# /nextjs - Next.js Application

$ARGUMENTS

---

## Task

This command guides Next.js App Router development using best practices.

### Steps:

1. **Clarify scope**
   - New app or feature update
   - App Router (default) vs Pages Router
   - Data fetching and caching needs

2. **Select core skills**
   - App Router patterns: `nextjs-app-router-patterns`
   - Next.js best practices: `nextjs-best-practices`
   - Performance patterns: `vercel-react-best-practices`

3. **Plan architecture**
   - Server vs client components
   - Route and layout structure
   - Error, loading, and not-found boundaries

4. **Implement**
   - Data fetching with cache strategy
   - Server actions where appropriate
   - Metadata and SEO essentials

5. **Verify**
   - Run lint and type checks
   - Validate key routes and data flows

---

## Usage Examples

```
/nextjs build a marketing site with app router
/nextjs add server actions for a form
/nextjs optimize bundle size and loading
/nextjs add parallel routes for dashboard
```
